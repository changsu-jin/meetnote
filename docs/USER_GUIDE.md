# MeetNote 사용자 매뉴얼

## 소개

MeetNote는 Obsidian에서 회의를 녹음하면 자동으로 전사, 화자 구분, 요약하여 마크다운 회의록을 생성하는 플러그인입니다.

**특징:**
- 완전 로컬 처리 (데이터가 외부로 전송되지 않음)
- 비용 0원 (유료 API 없음)
- 오프라인 동작 (최초 모델 다운로드 후)
- Apple Silicon GPU 가속

## 설치

### 자동 설치 (권장)

```bash
git clone https://github.com/changsu-jin/meetnote.git
cd meetnote
bash install.sh
```

설치 스크립트가 안내에 따라 진행하면 됩니다:
- Python 환경 자동 설정 (venv + 의존성 + MLX Whisper)
- 플러그인 빌드 및 vault 설치
- HuggingFace 토큰 입력 (화자 구분용, 무료 가입)
- Obsidian vault 경로 설정 (자동 탐색 또는 수동 입력)
- BlackHole 설치 여부 선택 (macOS, 화상회의 녹음 시 필요)

### HuggingFace 토큰 발급 방법

설치 중 HuggingFace 토큰을 묻습니다. 아래 순서로 발급하세요:

1. [huggingface.co](https://huggingface.co/join)에 가입 (무료)
2. [pyannote 모델 페이지](https://huggingface.co/pyannote/speaker-diarization-3.1)에서 "Agree" 클릭
3. [토큰 설정 페이지](https://huggingface.co/settings/tokens)에서 "New token" → 이름 입력 → 생성
4. 생성된 `hf_...` 토큰을 설치 스크립트에 붙여넣기

### 화상회의 녹음 설정 (선택)

대면 회의에서는 마이크만 사용하므로 이 설정이 **불필요**합니다.
Zoom/Teams/Meet 녹음이 필요한 경우에만:

1. 설치 스크립트에서 BlackHole 설치 선택
2. macOS "오디오 MIDI 설정" 앱 → "+" → "다중 출력 기기" 생성
3. "BlackHole 2ch"와 기본 스피커를 모두 체크
4. 시스템 사운드 출력을 이 기기로 변경

## 첫 실행

### 1. 플러그인 설정

플러그인을 처음 활성화하면 온보딩 위저드가 나타납니다.

- **백엔드 경로** (필수): meetnote/backend 디렉토리의 절대 경로 입력 (install.sh 완료 시 경로가 표시됩니다)
- "저장 후 시작" 클릭 → 사이드패널이 자동으로 열림

수동으로 설정하려면: Obsidian 설정 → MeetNote

**기본 설정 탭:**
| 항목 | 설명 |
|------|------|
| 백엔드 경로 * | Python 백엔드 디렉토리 (필수) |
| 후처리 모드 | 큐 모드(권장) 또는 즉시 처리 |
| 참석자 자동완성 경로 | vault 내 사용자 정보 폴더 |
| 발신자 이메일 | 회의록 이메일 전송 시 From 주소 |
| GitLab 링크 포함 | 이메일에 문서 GitLab URL 자동 포함 |
| 자동 태그/링크 | 요약에서 태그 추출 + 연관 회의 링크 |

**고급 설정 탭:**
| 항목 | 설명 |
|------|------|
| 서버 URL | WebSocket 주소 (기본값 권장) |
| Whisper 모델 | 음성 인식 모델 크기 (large-v3-turbo 권장) |
| HuggingFace 토큰 | 화자 분리용 토큰 |
| 화자 수 | 최소/최대 화자 수 (비워두면 자동) |
| Slack 연동 | Webhook URL + 테스트 |
| 보안 | 녹음 암호화, 자동 삭제 |

### 2. 서버 시작

사이드패널 헤더에서 **"시작"** 버튼 클릭 (첫 실행 시 모델 다운로드로 수 분 소요, 이후 약 10초)

상태 표시:
- 🟢 녹색 점: 서버 실행 중
- 🔴 빨간 점: 서버 중지

> 또는 터미널에서:
> ```bash
> cd meetnote/backend
> source venv/bin/activate
> python3 server.py
> ```

## 기본 사용법

### 녹음 및 회의록 생성 (큐 모드)

1. **문서 열기** — Obsidian에서 회의록을 기록할 마크다운 문서를 엽니다
2. **녹음 시작** — 사이드패널 헤더의 🎙 버튼 또는 리본 아이콘 클릭
3. **회의 진행** — 녹음이 자동으로 WAV 파일에 저장됩니다
4. **녹음 중지** — ⏹ 버튼 클릭 → "녹음 저장 완료" 알림
5. **처리 실행** — 사이드패널 "대기 중" 섹션에서 **"처리"** 버튼 클릭
6. **처리 완료** — 문서가 자동으로 열리고 처리 결과 알림 표시

> 큐 모드에서는 녹음과 처리를 분리하여, 여러 회의를 연속 녹음한 후 나중에 일괄 처리할 수 있습니다.

### 처리 진행 상태

처리 중에는 사이드패널에 실시간 진행 상태가 표시됩니다:
- 전사 중 → 화자 분리 중 → 음성 특징 추출 → 교정 중 → 요약 중

상태바에도 단계명과 퍼센트가 표시됩니다.

## 사이드패널 가이드

사이드패널은 4개 섹션으로 구성되며, 각 섹션은 접기/펼치기가 가능합니다.

### 대기 중

처리 대기 중인 녹음 목록입니다.
- **처리**: 전사 + 화자 구분 + 요약 실행
- **삭제**: 녹음 파일 + 메타데이터 + 연결 문서 모두 삭제

### 최근 회의

처리 완료된 회의 목록입니다.
- 문서 이름 클릭 → 해당 문서 열기
- **참석자**: 해당 회의의 화자 매핑 관리 (아래 참조)
- **재처리**: 메타 초기화 후 처음부터 다시 처리
- ⚠ 미등록 화자가 있으면 표시됨

### 회의 참석자

선택한 회의의 참석자를 관리합니다. "최근 회의"에서 **참석자** 버튼을 눌러 회의를 선택합니다.

**음성 인식 참석자:**
- 화자 구분으로 감지된 참석자
- "화자1", "화자2" → 실명 입력 후 **"음성 참석자 저장"**
- ✓ 표시: Speaker DB에 등록된 화자 (다음 회의에서 자동 인식)
- 이름 옆 (이메일) 표시

**수동 추가 참석자:**
- 화자 구분에서 감지되지 않은 참석자를 수동 추가
- 이름 + 이메일 입력 → **"추가"**

**이메일 전송:**
- 체크박스로 수신자 선택 → **"선택한 참석자에게 회의록 전송"**
- 설정에서 발신자 이메일이 필요합니다
- sendmail 사용 (별도 SMTP 설정 불필요)

### 음성 등록 사용자

Speaker DB에 등록된 모든 화자 목록입니다. 등록된 화자는 다음 회의에서 음성 유사도 기반으로 **자동 인식**됩니다.

- 검색: 이름/이메일로 필터링
- 수정: 이름, 이메일 변경
- 삭제: DB에서 제거 (이후 자동 인식 불가)

> 화자 등록은 "회의 참석자" 섹션에서 이름을 저장하면 자동으로 Speaker DB에 음성과 함께 등록됩니다.

## 추가 기능

### 자동 태그 및 연관 회의 링크

회의록 처리 시 자동으로:
- 요약에서 키워드를 추출하여 frontmatter 태그 생성
- vault 내 다른 회의록과 태그가 2개 이상 겹치면 "연관 회의" 섹션에 [[링크]] 추가
- 양방향 링크 — Graph view에서 회의 간 관계 시각화

설정에서 비활성화 가능.

### 문서 이동 시 메타 동기화

회의록 문서를 vault 내에서 이동하거나 이름을 변경하면, 백엔드의 메타 정보가 자동으로 갱신됩니다 (서버 실행 중일 때).

### 과거 회의 검색

명령어 팔레트 → "과거 회의 검색" → 질문 입력

예: "지난 달 API 성능 이슈 관련 논의"

관련 회의록을 찾아 LLM이 답변을 생성합니다.

### 회의 트렌드 대시보드

사이드패널 헤더의 📊 버튼 또는 명령어 팔레트 → "회의 트렌드 대시보드"

vault 내 모든 회의록(`type: meeting`)을 분석하여 통계 대시보드를 생성합니다:
- 총 회의 수, 시간, 결정사항, 액션아이템 완료율
- 월별 추이
- 주요 주제 (태그 빈도)
- 참석자 빈도
- 최근 회의 목록

### Slack 전송

1. Slack 앱에서 [Incoming Webhook](https://api.slack.com/messaging/webhooks) 생성
2. MeetNote 고급 설정 → Slack 연동 → 활성화 + Webhook URL 입력
3. "연결 테스트" 버튼으로 확인
4. 즉시 처리 모드에서 회의 완료 시 자동 전송

### 녹음 파일 암호화

1. MeetNote 고급 설정 → 보안 → "녹음 파일 암호화" 활성화
2. (선택) "자동 삭제" 기간 설정 (예: 30일)
3. 녹음 파일이 AES 암호화되어 저장, 원본 WAV는 삭제됨

## 설정 (config.yaml)

| 항목 | 기본값 | 설명 |
|------|--------|------|
| `whisper.model_size` | `large-v3-turbo` | STT 모델 (tiny/base/small/medium/large-v3/large-v3-turbo) |
| `whisper.language` | `ko` | 전사 언어 |
| `whisper.initial_prompt` | (회의 키워드) | 도메인 키워드 힌트 (고유명사 정확도 향상) |
| `audio.chunk_duration` | `30` | 실시간 전사 청크 크기 (초) |
| `audio.device` | `null` | 오디오 입력 (null=기본 마이크) |
| `diarization.min_speakers` | `null` | 최소 화자 수 (null=자동) |
| `diarization.max_speakers` | `null` | 최대 화자 수 (null=자동) |
| `speaker_db.similarity_threshold` | `0.70` | 화자 매칭 유사도 임계값 |
| `summary.timeout` | `120` | 요약 생성 타임아웃 (초) |

## 문제 해결

### 자주 묻는 질문

**Q: 인터넷 없이 사용 가능한가요?**
A: 네. 최초 1회 모델 다운로드 후 완전 오프라인 동작합니다. 단, AI 요약은 Claude CLI 또는 Ollama가 로컬에 설치된 경우에만 동작합니다.

**Q: 비용이 발생하나요?**
A: 아니요. 모든 처리가 로컬에서 이루어지며 유료 API를 사용하지 않습니다. (Claude Code Max 구독자는 요약에 Claude CLI 활용 가능 — 구독료에 포함)

**Q: 어떤 회의에서 사용 가능한가요?**
A: 대면 회의(마이크), 화상회의(BlackHole로 시스템 오디오 캡처), 전화 회의 모두 가능합니다.

**Q: 최소 시스템 사양은?**
A: macOS 12+ (Apple Silicon 권장). Intel Mac에서도 동작하나 CPU 처리로 속도가 느립니다. RAM 8GB 이상, 디스크 5GB 이상 권장.

**Q: 화자를 몇 명까지 구분할 수 있나요?**
A: 기본 2~6명 자동 감지. 고급 설정에서 `최대 화자 수`를 조정하여 더 많은 화자도 가능하나, 정확도가 떨어질 수 있습니다.

**Q: 화자 자동 인식이 안 됩니다**
A: 화자 매칭 유사도 임계값이 0.70입니다. 짧은 녹음이나 환경이 크게 다른 경우 유사도가 낮아 매칭이 안 될 수 있습니다. 서버 로그에서 유사도 수치를 확인할 수 있습니다.

**Q: 요약이 생성되지 않습니다**
A: Claude CLI(`claude` 명령어) 또는 Ollama가 설치되어 있어야 합니다. 서버 로그에서 `Summarizer ready (engines: ...)` 를 확인하세요. `['none']`이면 요약 엔진이 없는 것입니다.

자세한 문제 해결은 [TROUBLESHOOTING.md](TROUBLESHOOTING.md)를 참고하세요.

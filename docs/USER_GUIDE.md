# MeetNote 사용자 매뉴얼

## 소개

MeetNote는 Obsidian에서 회의를 녹음하면 자동으로 전사, 화자 구분, 요약하여 마크다운 회의록을 생성하는 시스템입니다. 서버(Docker)와 플러그인(Obsidian)으로 구성됩니다.

**특징:**
- 로컬 서버에서 음성 처리 (데이터가 외부로 전송되지 않음)
- 비용 0원 (유료 API 없음)
- Docker 기반 간편 설치
- Apple Silicon GPU 가속

## 아키텍처 개요

```
┌─────────────────────┐         WebSocket          ┌────────────────────────┐
│  Obsidian Plugin    │  ◀──────────────────────▶  │  MeetNote Server       │
│  (BRAT 설치)         │    오디오 스트림 / 결과      │  (Docker 컨테이너)       │
│                     │                            │                        │
│  - 마이크 캡처        │                            │  - Whisper 전사         │
│  - 요약 (Claude/     │                            │  - 화자 구분             │
│    Ollama/skip)     │                            │  - Speaker DB           │
│  - 이메일 전송 요청   │                            │  - SMTP 이메일           │
└─────────────────────┘                            └────────────────────────┘
```

## 설치

### 1. 서버 설치 (Docker)

```bash
git clone https://github.com/changsu-jin/meetnote-deployment.git
cd meetnote-deployment/backend
cp .env.example .env
# .env 파일을 편집하여 필요한 값 설정 (아래 참조)
docker compose up -d
```

첫 실행 시 Docker 이미지 빌드에 수 분이 소요됩니다 (모델 다운로드 포함). 이후 재시작은 수 초 이내.

#### .env 설정

| 항목 | 필수 | 설명 |
|------|------|------|
| `API_KEY` | 선택 | 원격 서버 인증용 키 (로컬 사용 시 불필요) |
| `SMTP_HOST` | 선택 | SMTP 서버 (기본: smtp.gmail.com) |
| `SMTP_PORT` | 선택 | SMTP 포트 (기본: 587) |
| `SMTP_USER` | 선택 | SMTP 로그인 계정 |
| `SMTP_PASSWORD` | 선택 | SMTP 앱 비밀번호 |
| `SMTP_USE_TLS` | 선택 | TLS 사용 여부 (기본: true) |
| `WHISPER_MODEL` | 선택 | STT 모델 (기본: large-v3-turbo) |
| `ENCRYPTION_ENABLED` | 선택 | 녹음 파일 암호화 (기본: false) |
| `AUTO_DELETE_DAYS` | 선택 | 녹음 자동 삭제 일수 (기본: 0 = 비활성) |

> 이메일 전송을 사용하려면 SMTP 관련 항목을 모두 설정하세요. 발신자 이메일(From)은 플러그인 설정에서 관리합니다.

#### 서버 관리 명령어

```bash
# 시작
docker compose up -d

# 로그 확인
docker compose logs -f

# 중지
docker compose down

# 재시작
docker compose restart
```

### 2. 플러그인 설치 (BRAT)

1. Obsidian 커뮤니티 플러그인에서 **BRAT** 설치 및 활성화
2. BRAT 설정 → "Add Beta plugin" → `changsu-jin/meetnote` 입력
3. Obsidian 설정 → 커뮤니티 플러그인 → **MeetNote** 활성화

> BRAT를 통해 설치하면 업데이트도 자동으로 관리됩니다.

### 3. 오디오 입력 설정

MeetNote 플러그인은 **Web Audio API**를 통해 마이크를 캡처합니다. Obsidian에서 최초 녹음 시 **마이크 권한 허용** 팝업이 나타나며, 반드시 허용해야 합니다.

#### 화상회의 녹음 (선택)

대면 회의에서는 마이크만 사용하므로 이 설정이 **불필요**합니다.
Zoom/Teams/Meet 녹음이 필요한 경우에만:

1. [BlackHole](https://existential.audio/blackhole/) 설치
2. macOS "오디오 MIDI 설정" 앱 → "+" → "다중 출력 기기" 생성
3. "BlackHole 2ch"와 기본 스피커를 모두 체크
4. 시스템 사운드 출력을 이 기기로 변경
5. 플러그인 설정에서 오디오 장치를 "BlackHole 2ch"로 선택

## 첫 실행

### 1. 플러그인 설정

Obsidian 설정 → MeetNote에서 아래 항목을 확인합니다.

| 항목 | 설명 | 기본값 |
|------|------|--------|
| 서버 URL | MeetNote 서버 WebSocket 주소 | `ws://localhost:8765` |
| API Key | 서버 인증 키 (.env의 API_KEY와 동일) | (비어있으면 인증 없음) |
| 오디오 장치 | 녹음에 사용할 마이크 | 시스템 기본 마이크 |
| 자동 태그/링크 | 요약에서 태그 추출 + 연관 회의 링크 | 활성화 |
| 참석자 자동완성 경로 | vault 내 사용자 정보 폴더 경로 | (비어있음) |
| 발신자 이메일 | 회의록 이메일 전송 시 From 주소 | (비어있음) |
| GitLab 링크 포함 | 이메일에 문서 GitLab URL 자동 포함 | 비활성화 |

### 2. 서버 상태 확인

사이드패널 헤더에서 서버 연결 상태를 확인합니다:
- 🟢 녹색 점: 서버 연결됨
- 🔴 빨간 점: 서버 미연결

서버가 연결되지 않으면 Docker 컨테이너가 실행 중인지 확인하세요:

```bash
docker compose ps
docker compose logs -f
```

## 기본 사용법

### 녹음 및 회의록 생성

1. **문서 열기** — Obsidian에서 회의록을 기록할 마크다운 문서를 엽니다
2. **녹음 시작** — 사이드패널 헤더의 🎙 버튼 또는 리본 아이콘 클릭 (최초 실행 시 마이크 권한 허용 필요)
3. **회의 진행** — 오디오가 Web Audio API로 캡처되어 서버로 스트리밍됩니다
4. **녹음 중지** — ⏹ 버튼 클릭 → "녹음 저장 완료" 알림
5. **처리 실행** — 사이드패널 "대기 중" 섹션에서 **"처리"** 버튼 클릭
6. **처리 완료** — 문서가 자동으로 열리고 처리 결과 알림 표시

> 녹음과 처리를 분리하여, 여러 회의를 연속 녹음한 후 나중에 일괄 처리할 수 있습니다.

### 요약 엔진

처리 실행 시 요약 단계에서 사용할 엔진을 선택할 수 있습니다:

| 엔진 | 설명 |
|------|------|
| **Claude CLI** | `claude` 명령어가 설치된 경우 사용 (Claude Code Max 구독 포함) |
| **Ollama** | 로컬 Ollama 서버가 실행 중인 경우 사용 |
| **skip** | 요약 생성을 건너뜀 (녹취록만 생성) |

> 요약은 플러그인 측에서 실행됩니다. 서버는 전사와 화자 구분만 담당합니다.

### 처리 진행 상태

처리 중에는 사이드패널에 실시간 진행 상태가 표시됩니다:
- 전사 중 → 화자 분리 중 → 음성 특징 추출 → 교정 중 → 요약 중

상태바에도 단계명과 퍼센트가 표시됩니다.

## 사이드패널 가이드

사이드패널은 4개 섹션으로 구성되며, 각 섹션은 접기/펼치기가 가능합니다.

### 대기 중

처리 대기 중인 녹음 목록입니다.
- **처리**: 전사 + 화자 구분 + 요약 실행
- **삭제**: 녹음 파일 + 메타데이터 + 연결 문서 모두 삭제

### 최근 회의

처리 완료된 회의 목록입니다.
- 문서 이름 클릭 → 해당 문서 열기
- **참석자**: 해당 회의의 화자 매핑 관리 (아래 참조)
- **재처리**: 메타 초기화 후 처음부터 다시 처리
- ⚠ 미등록 화자가 있으면 표시됨

### 회의 참석자

선택한 회의의 참석자를 관리합니다. "최근 회의"에서 **참석자** 버튼을 눌러 회의를 선택합니다.

**음성 인식 참석자:**
- 화자 구분으로 감지된 참석자
- "화자1", "화자2" → 실명 입력 후 **"음성 참석자 저장"**
- ✓ 표시: Speaker DB에 등록된 화자 (다음 회의에서 자동 인식)
- 이름 옆 (이메일) 표시

**수동 추가 참석자:**
- 화자 구분에서 감지되지 않은 참석자를 수동 추가
- 이름 + 이메일 입력 → **"추가"**

**이메일 전송:**
- 체크박스로 수신자 선택 → **"선택한 참석자에게 회의록 전송"**
- 플러그인 설정에서 발신자 이메일(From)을 입력해야 합니다
- 서버의 .env에 SMTP 설정이 필요합니다

### 음성 등록 사용자

Speaker DB에 등록된 모든 화자 목록입니다. 등록된 화자는 다음 회의에서 음성 유사도 기반으로 **자동 인식**됩니다.

- 검색: 이름/이메일로 필터링
- 수정: 이름, 이메일 변경
- 삭제: DB에서 제거 (이후 자동 인식 불가)

> 화자 등록은 "회의 참석자" 섹션에서 이름을 저장하면 자동으로 Speaker DB에 음성과 함께 등록됩니다.

## 추가 기능

### 자동 태그 및 연관 회의 링크

회의록 처리 시 자동으로:
- 요약에서 키워드를 추출하여 frontmatter 태그 생성
- vault 내 다른 회의록과 태그가 2개 이상 겹치면 "연관 회의" 섹션에 [[링크]] 추가
- 양방향 링크 — Graph view에서 회의 간 관계 시각화

플러그인 설정에서 비활성화 가능.

### 문서 이동 시 메타 동기화

회의록 문서를 vault 내에서 이동하거나 이름을 변경하면, 서버의 메타 정보가 자동으로 갱신됩니다 (서버 연결 중일 때).

### 과거 회의 검색

명령어 팔레트 → "과거 회의 검색" → 질문 입력

예: "지난 달 API 성능 이슈 관련 논의"

관련 회의록을 찾아 LLM이 답변을 생성합니다.

### 회의 트렌드 대시보드

사이드패널 헤더의 📊 버튼 또는 명령어 팔레트 → "회의 트렌드 대시보드"

vault 내 모든 회의록(`type: meeting`)을 분석하여 통계 대시보드를 생성합니다:
- 총 회의 수, 시간, 결정사항, 액션아이템 완료율
- 월별 추이
- 주요 주제 (태그 빈도)
- 참석자 빈도
- 최근 회의 목록

### 녹음 파일 암호화

서버 .env에서 설정합니다:
- `ENCRYPTION_ENABLED=true` — 녹음 파일이 AES 암호화되어 저장
- `AUTO_DELETE_DAYS=30` — 30일 후 자동 삭제 (0이면 비활성)

## 문제 해결

### 자주 묻는 질문

**Q: 인터넷 없이 사용 가능한가요?**
A: 서버가 로컬에서 실행 중이면 전사와 화자 구분은 오프라인으로 동작합니다. AI 요약은 Claude CLI 또는 Ollama가 로컬에 설치된 경우에만 동작합니다.

**Q: 비용이 발생하나요?**
A: 아니요. 모든 처리가 로컬에서 이루어지며 유료 API를 사용하지 않습니다. (Claude Code Max 구독자는 요약에 Claude CLI 활용 가능 — 구독료에 포함)

**Q: 어떤 회의에서 사용 가능한가요?**
A: 대면 회의(마이크), 화상회의(BlackHole로 시스템 오디오 캡처), 전화 회의 모두 가능합니다.

**Q: 최소 시스템 사양은?**
A: Docker가 실행 가능한 환경이면 됩니다. 서버는 macOS (Apple Silicon 권장), Linux를 지원합니다. RAM 8GB 이상, 디스크 10GB 이상 권장.

**Q: 화자를 몇 명까지 구분할 수 있나요?**
A: 기본 2~6명 자동 감지. 서버 설정에서 조정 가능하나, 화자가 많으면 정확도가 떨어질 수 있습니다.

**Q: 화자 자동 인식이 안 됩니다**
A: 화자 매칭 유사도 임계값이 0.70입니다. 짧은 녹음이나 환경이 크게 다른 경우 유사도가 낮아 매칭이 안 될 수 있습니다. 서버 로그(`docker compose logs -f`)에서 유사도 수치를 확인할 수 있습니다.

**Q: 요약이 생성되지 않습니다**
A: 요약은 플러그인 측에서 실행됩니다. Claude CLI(`claude` 명령어) 또는 Ollama가 로컬에 설치되어 있어야 합니다. 둘 다 없으면 "skip" 모드로 녹취록만 생성됩니다.

**Q: 마이크 권한을 실수로 거부했습니다**
A: macOS의 경우 시스템 설정 → 개인 정보 보호 및 보안 → 마이크에서 Obsidian을 허용하세요. 이후 Obsidian을 재시작합니다.

**Q: 이메일 전송이 안 됩니다**
A: 서버 .env에 SMTP 설정(SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASSWORD)이 완료되었는지, 플러그인 설정에서 발신자 이메일이 입력되었는지 확인하세요.

자세한 문제 해결은 [TROUBLESHOOTING.md](TROUBLESHOOTING.md)를 참고하세요.
